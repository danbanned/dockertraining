# Live Demo Checklist

## Before the Demo
- Confirm the app runs locally.
- Confirm the database is reachable.
- Confirm at least one AI provider is configured.
- Confirm login works and sets the `brightpath_token` cookie.
- Confirm roadmap, tasks, jobs, and one AI feature load successfully.
- Confirm GitHub Actions workflow is visible and recent pipeline history is available.
- Prepare one small UI change for the push-to-deploy section.

## During the Demo
- Introduce the system architecture.
- Show the main user flow.
- Explain backend logic and AI fallback.
- Show the CI/CD workflow file.
- Make the small code change.
- Push the change.
- Show the pipeline running.
- Verify the change after deployment.

## After the Demo
- Keep links ready for the repo, workflow run, and deployed app.
- Be ready to explain one bug you fixed and how you fixed it.
